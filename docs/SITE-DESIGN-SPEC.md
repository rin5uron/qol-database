## ページ一覧

### 1. トップページ (`index.html`)

**目的**: サイトの顔、記事への導線

**構成**:
1. ヒーローセクション（りんちゃん + キャッチコピー）
2. コンセプトセクション（「こんなお悩み、ありませんか？」）
3. リンゴラボのこと（記事・商品・アプリの3軸）
4. 記事セクション（6記事表示 + 「もっと見る」）
5. プロフィールセクション
6. フッター

**記事表示**:
- PC: 3列グリッド
- スマホ: 2列グリッド
- 表示数: 6記事
- ボタン: 「もっと見る」→ `/articles.html`

---

### 2. 記事一覧ページ (`articles.html`)

**目的**: 全記事を一覧表示、カテゴリフィルター

**構成**:
1. ページタイトル「記事一覧」
2. カテゴリフィルター（温活・食事・美容・すべて）
3. 記事グリッド（12記事/ページ）
4. ページネーション

**レイアウト**:
- PC: 3列 × 4行 = 12記事
- スマホ: 2列 × 6行 = 12記事

**カテゴリフィルター**:
```html
<nav class="category-filter">
  <a href="articles.html" class="active">すべて</a>
  <a href="articles-onkatsu.html">温活</a>
  <a href="articles-food.html">食事</a>
  <a href="articles-beauty.html">美容</a>
</nav>
```

## URL設計

### URL命名規則

**原則**:
- 小文字
- ハイフン区切り
- 英語表記
- 短く分かりやすく

### URL一覧

| ページ | URL | 備考 |
|--------|-----|------|
| トップ | `/` | index.html |
| 記事一覧 | `/articles.html` | すべての記事 |
| 温活カテゴリ | `/articles-onkatsu.html` | 温活記事のみ |
| 食事カテゴリ | `/articles-food.html` | 食事記事のみ |
| 美容カテゴリ | `/articles-beauty.html` | 美容記事のみ |
| LAVA記事 | `/article-lava-review.html` | 体験レビュー |
| よもぎ蒸し記事 | `/article-yomogi-review.html` | 体験レビュー |
| 無添つくりおき記事 | `/article-chef-tsukurioki.html` | 商品解説 |
| 商品ページ | `/product-tea.html` | 準備中 |
| アプリページ | `/app-health.html` | 準備中 |

### 記事URL命名パターン

**体験レビュー**:
- `article-{サービス名}-review.html`
- 例: `article-lava-review.html`

**ハウツー**:
- `article-{トピック}-howto.html`
- 例: `article-onkatsu-howto.html`

**商品解説**:
- `article-{商品名}-guide.html`
- 例: `article-chef-tsukurioki.html`

## ページ遷移・導線

### トップページからの導線

```
トップページ
├── ヒーロー「記事を読む」→ articles.html
├── 記事セクション
│   ├── 記事カード → 記事詳細
│   └── 「もっと見る」→ articles.html
└── ナビゲーション「記事を読む」→ articles.html
```

### 記事一覧ページの導線

```
記事一覧ページ
├── カテゴリフィルター
│   ├── 温活 → articles-onkatsu.html
│   ├── 食事 → articles-food.html
│   └── 美容 → articles-beauty.html
├── 記事カード → 記事詳細
└── ページネーション → 次のページ
```

### 記事詳細ページの導線

```
記事詳細ページ
├── パンくずリスト
│   └── トップ > カテゴリ > 記事
├── 本文中のアフィリエイトリンク → 外部サイト
├── プロフィールカード → プロフィールセクション
└── 関連記事 → 他の記事詳細
```

| 要素 | スマホ | タブレット | PC |

| 記事カード | 2列 | 2列 | 3列 |

- 最小サイズ: 44×44px（iOS基準）

## パフォーマンス要件

### ページ読み込み速度

- **LCP（Largest Contentful Paint）**: 2.5秒以内
- **INP（Interaction to Next Paint）**: 200ms以内
- **CLS（Cumulative Layout Shift）**: 0.1以内

### 画像最適化

- **フォーマット**: WebP（フォールバックJPG）
- **圧縮**: 品質80-85%
- **遅延読み込み**: `loading="lazy"`
- **レスポンシブ**: `srcset`で複数サイズ

### CSS/JS

- **CSS**: インライン化（クリティカルCSS）
- **JS**: 非同期読み込み（`async`/`defer`）
- **Tailwind**: CDN（本番環境では最適化版）
